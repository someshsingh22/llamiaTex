\textbf{An agent} \(\mathcal{A}\) maximizes a performance measure based on experience \cite{russell1995artificial}. It operates within an MDP \((S, A, P, R, \gamma)\), where \(S\) is the state space, \(A\) the action space, \(P\) the transition dynamics, \(R\) the reward function, and \(\gamma\) the discount factor. The agent's policy \(\pi: S \rightarrow \Delta(A)\) maps states to action distributions, selecting actions \(a_t\) at each step \(t\) to transition from \(s_t\) to \(s_{t+1}\) while receiving reward \(r_t\) \cite{puterman2014markov,sutton1999reinforcement}. Agents like Stockfish~\cite{maharaj2022chess}, AlphaZero~\cite{silver2017mastering} and Leela Chess Zero (LC0)~\cite{monroe2024mastering} model their policy and value estimates using encoder backbones \cite{nasu2018efficiently} which maps \(s\in S\) to a latent embedding \(\vh_{s}\).

\textbf{LLM} denoted as \(F_{\theta}\) are parameterized by $\theta$ and trained on large-scale textual data \(\sV\), with their token output distribution expressed autoregressively as
\(
p_{\theta}(w_{t} \mid w_{1\ldots t-1}) = F_{\theta}\bigl(w_{t}\;\big|\;w_{1\ldots t-1}\bigr).
\) where $w_i \in \sV$. Beyond text generation, recent research has increasingly positioned LLMs as reasoning and decision-making agents, capable of planning, self-reflection, and interaction with external tools \cite{liu2023agentbench}. Below, we formalize the key frameworks that extend LLMs toward autonomous agency.

\textbf{Design of LLM based Autonomous Agents} 
Modern LLM-based agents are designed to operate over token sequences that unify reasoning and action within a single autoregressive framework. Unlike traditional MDP-based agents that map states to discrete actions, LLM agents generate natural language ``Think'' tokens and structured ``Act'' tokens to represent internal deliberation and external operations. Reasoning steps may take the form of chain-of-thought traces \cite{wei2022chain}, collaborative dialogue among agents (AutoGen \cite{wu2023autogen}), interleaved reasoning-action sequences (ReAct \cite{yao2022react}), or multimodal end-to-end outputs (Unified IO2 \cite{lu2024unified}). Tool use is incorporated through explicit sub-agent or API calls, as seen in Toolformer \cite{schick2023toolformerlanguagemodelsteach}, HuggingGPT \cite{shen2024hugginggpt}, and surveys of tool-augmented models \cite{mialon2023augmented,ma2024sciagenttoolaugmentedlanguagemodels}. These systems enable LLMs to invoke and condition on external function outputs, forming a tight loop between thought, action, and environment feedback.

The agent’s token sequence \(\tau = (w_1, w_2, \dots, w_T)\) thus embeds both the reasoning trace and any tool-assisted decisions, while prior tokens and sub-agent observations define the context for future decisions. Over time, the agent’s internal chain-of-thought tokens \(\tau_{<t}\) and retrieved outputs (e.g., from tools or sub-agents \(\mathcal{A}(s)\)) together shape its policy. To align these behaviors with task goals, the agent is optimized with respect to a reward function \(\mathcal{R}(s, \tau, \text{prompt})\) that evaluates the token trajectory \(\tau\) under a given task state and prompt. This leads to the general training objective:
\vspace{-2mm}
\begin{equation}
\max_\theta \; \mathbb{E}_{s \sim \mathcal{D}} \Bigl[
  \mathbb{E}_{\tau \sim \pi_\theta(\cdot \mid s)} \bigl[\mathcal{R}(s, \tau, \text{prompt})\bigr]
\Bigr],  
\label{eq:llm_agent_objective}
\end{equation}
where \(\mathcal{D}\) is a distribution over tasks or environments, and \(\pi_\theta\) is the LLM-induced token policy. This framework accommodates a range of downstream supervision signals—such as explanation quality, success rates, or user satisfaction—while allowing flexible interleaving of language, reasoning, and action. As demonstrated in recent work on generative simulation \cite{sun2024factorsim}, autonomous driving \cite{hu2024agentscodriver}, and commonsense reasoning \cite{zhao2024large}, LLM-based agents provide a generalizable interface for decision-making. Optimization techniques often incorporate iterative self-reflection (Reflexion \cite{shinn2023reflexionlanguageagentsverbal}) and affordance-aware grounding (SayCan \cite{ahn2022can}) to enhance long-term planning and alignment.



Recent work suggests that reasoning in large language models does not have to be expressed explicitly through token-level chains of thought. Instead, the models’ hidden states themselves encode structured semantic and procedural information that can support multi-step inference in a continuous space (Chen et al., 2025b; Hao et al., 2024; Liu et al., 2024; Zhang et al., 2025). Approaches such as CoCoNut (Hao et al., 2024), along with latent-space steering and editing methods (e.g., RepE (Zou et al., 2023), LoT (Fungwacharakorn et al., 2024)), show that directly manipulating internal representations can shape downstream reasoning behavior and improve controllability, without relying on explicit textual rationales. These methods operate by modifying latent trajectories that influence generation, enabling forms of reasoning that are difficult to realize through discrete token sequences alone. However, this line of work has largely focused on the internal states of a single language model. A separate line of research, \textit{latent multi-agent collaboration}~\cite{latentmas2025,hao2024training}, extends continuous-space reasoning to multiple models by replacing text-based communication between language models with continuous representation exchange---passing hidden states or KV caches rather than decoded tokens. While this enables richer collaboration than text, it requires homogeneous architectures: all participating models must share compatible embedding spaces. This precludes integration with pretrained non-language agents---such as a 240M-parameter game-playing transformer---whose representations occupy fundamentally different spaces from the LLM's. Similarly, \textit{static multimodal projection} methods~\cite{liu2023visual,li2023blip} encode an external input once and hold the resulting representation fixed throughout generation. In sequential decision domains, where the LLM's actions change the environment state, a single static encoding becomes stale and must be refreshed for continued reasoning. Neither approach addresses the setting we consider: coupling an LLM with a pretrained non-language agent whose state evolves dynamically during multi-step interaction.

\textbf{Internal Agents:} However, these approaches to integrate agents collapse the sub-agent's rich internal state representations into sparse symbolic outputs (e.g., $\mathcal{A}(s) \to \text{Nc2}$). 
This discards critical information encoded in the agent's latent state $\vh_s = G_\psi(s)$, which captures implicit planning signals, positional reasoning, and long-term value estimates as suggested by \citet{jenner2024evidence}. 


To solve this,  we propose to \emph{internalize} the agent by incorporating its latent state $\vh_s$ into the LLM. Formally, we update the objective \ref{eq:llm_agent_objective} to:  
\vspace{-2mm}
\begin{equation}
\max_\theta \; \mathbb{E}_{s \sim \mathcal{D}} \Bigl[
  \mathbb{E}_{\tau \sim \pi_\theta(\cdot \mid\vh_s)} \bigl[\mathcal{R}(s, \tau, \text{prompt})\bigr]
\Bigr],
\label{eq:llamia_agent_objective}
\end{equation}
