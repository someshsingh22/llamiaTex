
\section{Introduction}
\vspace{-2mm}

% =============================================================================
% P1: The landscape — two established paradigms for LLM-agent collaboration
% =============================================================================

Large language models increasingly serve as orchestrators of external agents and tools for planning, reasoning, and domain-specific decision-making~\cite{wu2023autogen,schick2023toolformerlanguagemodelsteach,shen2024hugginggpt,yao2022react}. Two well-established paradigms govern this collaboration. In \textit{tool calling}, the LLM invokes external agents through structured function interfaces, receives symbolic or textual responses, and conditions its next reasoning step on these outputs~\cite{schick2023toolformerlanguagemodelsteach,yao2022react,shen2024hugginggpt,nakano2021webgpt}. Reinforcement learning has emerged as a significant research direction for optimizing these invocation strategies, with recent work demonstrating substantial gains on mathematical reasoning and code generation~\cite{toolrl2025,guo2025deepseek}. In \textit{multi-agent conversation}, multiple agents coordinate through structured prompts, shared dialogue histories, or role-based exchanges~\cite{wu2023autogen,shinn2023reflexionlanguageagentsverbal}. \textit{Both paradigms share a fundamental constraint: all information exchanged between the LLM and its collaborating agents must pass through the bottleneck of natural language.}

% =============================================================================
% P2: Why this matters — the information bottleneck (bridge paragraph)
% =============================================================================

This constraint becomes consequential because many of the most capable agents in AI are not language models. Pretrained neural networks for strategic game play~\cite{silver2017mastering,monroe2024mastering}, robotic manipulation~\cite{brohan2023rt2}, reward estimation~\cite{xu2023imagereward}, and domain-specific evaluation encode their knowledge as continuous internal representations: probability distributions over actions, value estimates for future states, and learned features that capture domain-specific structure. These representations occupy fundamentally different spaces from the LLM's token embeddings. Under current paradigms, this rich continuous state must be compressed into a handful of symbolic tokens before the LLM can process it---and the LLM, in turn, must spend many more tokens parsing and reasoning about these sparse outputs to reconstruct information that already existed in the agent's latent space. In sequential decision-making, where each step requires a fresh agent query and a fresh compression, this information loss accumulates across the trajectory. This raises a natural question: \textit{can we bypass the text interface entirely and give the LLM direct access to the agent's internal representations?}

% =============================================================================
% P3: Making it concrete — what exactly is lost
% =============================================================================

To make this bottleneck concrete, consider integrating a pretrained game-playing agent with an LLM for collaborative analysis. The agent's encoder maps a game state to a latent embedding that encodes a full probability distribution over all ${\sim}$30 legal actions, scalar value estimates for the current and future positions, and learned positional features that capture long-term strategic structure---representations that recent mechanistic interpretability work has shown to encode genuine look-ahead and planning~\cite{mcgrath2022acquisition,jenner2024evidence}. Yet under standard text-based interfaces, even an enriched query returns only the top few actions with scalar evaluations---roughly 10--20 tokens, a compression ratio exceeding 1000:1 relative to the agent's representation dimensionality. The full policy distribution, the value landscape across positions, and the learned positional features are all discarded. The LLM then spends ${\sim}$100+ tokens parsing these sparse symbolic outputs. In a multi-step planning interaction---where the LLM must reason over a sequence of actions and their consequences---this compression-parsing cycle repeats at every step, compounding the information deficit across the trajectory.

% =============================================================================
% P4: Why existing solutions don't close this gap
% =============================================================================

Prior approaches address parts of this problem but leave the core gap open. \textit{Language-only fine-tuning}~\cite{feng2024chessgpt} trains LLMs on action sequences and textual annotations, operating entirely in token space without access to the agent's learned representations. The ceiling of this approach is the information content of text: empirically, a 13B model fine-tuned this way achieves only 11\% behavior cloning accuracy where our approach achieves 55\% (Table~\ref{tab:chess_benchmarks}). \textit{RL-augmented tool calling}~\cite{toolrl2025} teaches the LLM optimal invocation strategies---when to call the agent and what to ask---but the communication channel remains textual. No matter how effectively the LLM times its tool calls, the agent's response is still compressed to a handful of tokens; the information bottleneck persists regardless of invocation strategy.

% =============================================================================
% P5: Our approach — motivation first, then definition
% =============================================================================

The gap identified above---between the agent's rich continuous representations and the LLM's text-based reasoning---suggests a direct resolution: rather than requiring agents to verbalize their internal states as text, project those states directly into the LLM's reasoning process. If the LLM could attend to the agent's latent representation in the same way it attends to its own chain-of-thought tokens, the information bottleneck would be eliminated at the source.

We call this approach \textit{latent state internalization}. A lightweight adapter $H_\phi: \mathbb{R}^{d} \rightarrow \mathbb{R}^{K \times e}$ ({<}1\% additional parameters) maps the agent's latent state into $K{=}32$ continuous tokens in the LLM's embedding space. These tokens are interleaved with language and action tokens during autoregressive generation. Crucially, an \texttt{<InvokeAgent>} mechanism enables \textit{dynamic re-encoding}: when the LLM generates an action, the environment transitions, the agent re-encodes the new state, and fresh latent tokens are appended to the reasoning trace. This closed loop supports iterative planning over evolving states without reverting to the text interface. We train these augmented reasoning traces with Group Relative Policy Optimization (GRPO), enabling the model to learn \textit{when} to invoke the agent, \textit{which} configuration to use (routing between agents of different search depths), and \textit{how} to condition its downstream reasoning on continuous rather than symbolic representations.

% =============================================================================
% P6: Why chess — reframed around the task suite
% =============================================================================

We validate this paradigm on chess, chosen as a controlled testbed for three reasons. First, chess provides deterministic dynamics and a calibrated evaluation infrastructure---the Elo rating system, with 60+ years of standardized measurement and extensive human play data across every skill level. Second, superhuman pretrained agents exist (Leela Chess Zero~\cite{monroe2024mastering}, Stockfish~\cite{romstad2008stockfish}) whose internal representations have been studied through mechanistic interpretability~\cite{jenner2024evidence,mcgrath2022acquisition}, enabling controlled comparison of what is gained by internalizing their latent states versus querying them symbolically. Third, and most importantly, chess offers a rich suite of evaluation tasks that span both \textit{decision-making}---gameplay, behavior cloning across skill levels, and instruction-conditioned planning---and \textit{decision-understanding}---natural language game commentary and estimation of human-perceived difficulty and interest. Human experts already rely on engines as collaborative tools for all of these tasks; the question we address is whether internalizing the engine's representations enables more effective collaboration than the current text-based interface.

% =============================================================================
% P7: Key findings — separate from contributions
% =============================================================================

We instantiate this framework as \textbf{LLAMIA} (Large Language and Action Models with Internal Agents), combining Vicuna-7B/13B~\cite{zheng2023judging} with Leela Chess Zero's BT4 network (240M parameters). Our key empirical findings:

\begin{itemize}[leftmargin=*, topsep=2pt, itemsep=2pt]
    \item \textbf{Internalization outperforms tool calling across all five task categories.} A 13B model that has internalized the engine outperforms 30$\times$ larger LLMs that query the same engine through text: LLAMIA-13B achieves 0.61 G-eval on commentary vs.\ 0.52 for GPT-4o with Lc0 tool access, and a 17\% win-rate gain on instruction-conditioned planning vs.\ 10\% for LLaMA-3 405B with Lc0 (Table~\ref{tab:chess_benchmarks}).

    \item \textbf{A single model matches or exceeds task-specific specialists.} LLAMIA-13B achieves 55\% behavior cloning accuracy with one model vs.\ 53\% for Maia's 5-model ensemble trained on 12$\times$ more data, and 2279$\pm$41 Elo (International Master strength) without search, comparable to the 270M Transformer trained on 500$\times$ more positions (Table~\ref{tab:gameplay-performance}).

    \item \textbf{Dynamic re-encoding is the critical mechanism for multi-step reasoning.} Removing \texttt{<InvokeAgent>} causes 64\% degradation on conditional planning and 22\% on commentary, but only 1--4\% on single-step tasks (gameplay, move prediction). This asymmetry precisely isolates the contribution of iterative latent re-encoding to sequential reasoning (Table~\ref{tab:invoke_ablation}).

    \item \textbf{Both training stages are necessary and complementary.} Stage~1 (projector alignment on 50M state-policy pairs) alone achieves strong gameplay (2198 Elo) but poor language tasks (0.31 G-eval). Stage~2 without Stage~1 is unstable and plateaus 190 Elo below the full system. The staged approach leverages abundant synthetic data for stable initialization before instruction-tuning (Table~\ref{tab:stage_ablation}).
\end{itemize}

\vspace{-1mm}
\subsection{Contributions}
\vspace{-1mm}

\begin{enumerate}[leftmargin=*, labelsep=0.5em, topsep=2pt, itemsep=2pt]
    \item \textbf{Latent state internalization for pretrained non-language agents (Section~\ref{sec:methodology}):} We formalize and instantiate a paradigm for coupling LLMs with pretrained neural agents by projecting their internal representations into the LLM's reasoning trace via learned adapters. Unlike tool calling, this preserves the agent's continuous policy distributions, value estimates, and learned features without compressing them to text.

    \item \textbf{Dynamic re-encoding with \texttt{InvokeAgent} (Section~\ref{sec:methodology}):} We introduce a mechanism for iterative reasoning over evolving states: the LLM generates actions, transitions the environment, and receives fresh agent encodings mid-generation. Ablations confirm this is the critical differentiator---removing it causes 64\% degradation on multi-step planning but only 1--4\% on single-step tasks.

    \item \textbf{RL-trained agent-in-the-loop reasoning (Section~\ref{sec:methodology}):} We train augmented chain-of-thought traces with GRPO, enabling the model to learn optimal agent invocation strategies---including routing between multiple agent configurations of different depths. This optimizes over full reasoning rollouts containing latent tokens, rather than bare action prediction.

    \item \textbf{Comprehensive empirical validation (Section~\ref{sec:experiments}):} On chess as a controlled testbed, a single LLAMIA model matches or exceeds task-specific specialists across five evaluation categories spanning decision-making and decision-understanding, while outperforming 30$\times$ larger LLMs with tool access. We release LLAMIA-Bench for evaluating LLM-agent integration on sequential decision tasks.
\end{enumerate}
