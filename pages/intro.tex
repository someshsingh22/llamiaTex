
\section{Introduction}
\vspace{-2mm}

% =============================================================================
% P1: The setting — LLMs orchestrate agents, all communication is text
% =============================================================================

Large language models increasingly solve complex tasks by coordinating with specialized agents that provide domain-specific knowledge and environment interaction~\cite{wu2023autogen,schick2023toolformerlanguagemodelsteach,shen2024hugginggpt}. Two established paradigms support this coordination. In \textit{tool and function calling}, the LLM generates structured invocations, receives text responses from an external agent, and conditions its next reasoning step on this output~\cite{yao2022react,schick2023toolformerlanguagemodelsteach}. In \textit{multi-agent conversation}, LLM-based subagents collaborate through natural language dialogue, delegating subtasks and synthesizing results through verbal exchange~\cite{wu2023autogen,hong2023metagpt}. A growing body of work improves these paradigms---reinforcement learning for tool invocation strategy~\cite{toolrl2025}, structured prompting for multi-agent coordination~\cite{shinn2023reflexionlanguageagentsverbal}, affordance-aware grounding for embodied agents~\cite{ahn2022can}---yet all share a common design: the agent's output must be \textit{verbalized}, serialized into discrete text tokens, before the LLM can reason about it. \textit{When the agent's value lies in learned continuous representations rather than symbolic knowledge, what information is lost in this verbalization?}

% =============================================================================
% P2: Why this matters — the information bottleneck, motivated concretely
% =============================================================================

Many high-performance agents derive their strength not from symbolic rules but from learned continuous representations: action distributions, value functions, and state features acquired through large-scale training~\cite{silver2017mastering,monroe2024mastering,brohan2023rt2}. These representations encode far more than a single best answer---they capture the agent's assessment of every available action, its estimates of future outcomes under each, and abstract features of the current state that support long-horizon planning~\cite{mcgrath2022acquisition,jenner2024evidence}. When these representations are verbalized for an LLM, the vast majority of this information is discarded. For instance, a neural chess agent internally computes a full probability distribution over ${\sim}$30 legal actions, value estimates for future states, and learned positional features; through a standard text interface, this reduces to a handful of tokens (``best move: Nc2, evaluation: +0.3''). The consequences extend beyond a single query. The LLM cannot evaluate alternative actions without re-querying the agent, cannot explain the agent's reasoning about options it did not select, and cannot condition on the agent's uncertainty or confidence. In multi-step tasks---where the model must plan, act, observe, and re-plan---this loss compounds at every interaction, making the collaboration both \textit{informationally lossy} (the agent computes rich representations that are discarded) and \textit{computationally wasteful} (the LLM spends tokens reconstructing what it can from sparse text).

% =============================================================================
% P3: Why existing solutions don't close the gap
% =============================================================================

Existing approaches each address part of this problem but leave the core bottleneck intact. \textit{Language-only fine-tuning}~\cite{feng2024chessgpt} trains LLMs on move sequences and textual annotations, operating entirely in token space without access to the agent's learned representations; a 13B model fine-tuned this way achieves 11\% accuracy on predicting human behavior across skill levels, where our approach achieves 55\% (Table~\ref{tab:chess_benchmarks}). \textit{RL-augmented tool calling}~\cite{toolrl2025} teaches LLMs optimal invocation strategies through reinforcement learning, but the communication channel remains textual---the information bottleneck persists regardless of when or how the tool is called. \textit{Static multimodal projection}~\cite{liu2023visual,li2023blip} maps an external representation into the LLM's embedding space, but encodes it once and holds it fixed throughout generation; in sequential tasks where the LLM's actions change the environment state, this static encoding becomes stale after the first step.

% =============================================================================
% P4: Our approach — motivate each component, then introduce it
% =============================================================================

The information bottleneck is the verbalization step itself---the lossy conversion of continuous representations into text. A natural approach is to bypass it entirely, letting the LLM reason over the agent's internal representations directly. We study this through \textit{latent state internalization}: a lightweight projector $H_\phi: \mathbb{R}^{d} \rightarrow \mathbb{R}^{K \times e}$ ({<}1\% additional parameters) maps the agent's latent state into $K{=}32$ continuous tokens in the LLM's embedding space. These tokens are interleaved with language and action tokens during autoregressive generation, so the LLM reasons \textit{with} the agent's representations rather than \textit{about} their text summaries. Because the environment state changes after each action, a static encoding becomes invalid after the first step. To address this, we introduce an \texttt{<InvokeAgent>} re-invocation mechanism: when the LLM generates an action, the environment transitions, the agent re-encodes the new state, and fresh latent tokens are appended to the reasoning trace---creating a closed loop between language reasoning and agent perception. To learn not just \textit{when} to invoke the agent but \textit{how} to ground reasoning in the resulting continuous tokens, we train over full chain-of-thought rollouts containing latent tokens using Group Relative Policy Optimization (GRPO). This extends naturally to multi-agent settings: different agent configurations (varying in search depth and playing strength) are internalized simultaneously, and the LLM learns to route between them based on task demands.

% =============================================================================
% P5: Why chess — properly motivated
% =============================================================================

To rigorously evaluate whether internalization improves LLM-agent collaboration, we require a domain that satisfies three criteria. First, superhuman pretrained agents must exist with well-studied internal representations, so we can precisely measure what information transfers and quantify the gain from internalization. Second, the domain should be one where humans and AI \textit{already rely on agents as collaborative tools}---not merely as solvers---so that improving this collaboration has direct practical significance. In chess, commentators use engines to explain strategic ideas, players use them to prepare openings and analyze games, and analysts use them to evaluate positions at every level of play; the collaboration between human reasoning and engine computation is central to modern chess practice. Third, the domain must offer a breadth of evaluation tasks that test different collaboration capabilities: not just action selection (which engines already solve far beyond human level), but natural language explanation, human behavior prediction across skill levels, and instruction-following while leveraging the agent. Chess satisfies all three criteria. We pair Vicuna-7B/13B~\cite{zheng2023judging} with Leela Chess Zero's BT4 network~\cite{monroe2024mastering} (240M parameters), a neural chess agent whose internal representations have been shown to encode lookahead and planning signals~\cite{jenner2024evidence}.

% =============================================================================
% P6: Key results — grouped by insight, not a laundry list
% =============================================================================

We instantiate this framework as \textbf{LLAMIA} (Large Language and Action Models with Internal Agents). Across five evaluation categories, internalization consistently outperforms text-based collaboration:

\begin{itemize}[leftmargin=*, topsep=2pt, itemsep=2pt]
    \item \textbf{Higher accuracy across tasks:} LLAMIA-13B surpasses 30$\times$ larger LLMs with tool access on action selection (2279$\pm$41 Elo without search), natural language game explanation (0.61 vs.\ 0.52 quality score for GPT-4o with engine access), and strategic planning (17\% win-rate improvement vs.\ 10\% for LLaMA-3 405B with tool access) (Tables~\ref{tab:chess_benchmarks},~\ref{tab:gameplay-performance}).

    \item \textbf{Greater efficiency:} Each agent interaction uses $K{=}32$ continuous tokens, replacing the 100{+} verbalized tokens required by tool-calling pipelines, while preserving richer information from the agent's internal state.

    \item \textbf{New collaboration strategies:} Internalization enables instruction-conditioned planning (``play aggressively from this position'') and multi-agent routing (selecting among agents of different strengths based on position complexity)---capabilities that have no analogue in text-based systems.

    \item \textbf{Data efficiency:} A single LLAMIA model matches the accuracy of task-specific specialists trained on 12--500$\times$ more data, including a 5-model ensemble for human behavior prediction (55\% vs.\ 53\%) and a 270M dedicated agent for gameplay (2279 vs.\ 2299 Elo) (Table~\ref{tab:gameplay-performance}).
\end{itemize}

\noindent Controlled ablations isolate the source of these gains. Removing the re-invocation mechanism degrades performance on multi-step planning by 64\% and on explanation by 22\%, but affects single-step action prediction by only 1--4\% (Table~\ref{tab:invoke_ablation}). This asymmetry confirms that iterative access to the agent's evolving representations---not merely a one-time encoding---is the critical factor for multi-step reasoning tasks.

\vspace{-1mm}
\subsection{Contributions}
\vspace{-1mm}

\begin{enumerate}[leftmargin=*, labelsep=0.5em, topsep=2pt, itemsep=2pt]
    \item \textbf{Latent state internalization (Section~\ref{sec:methodology}):} We formalize and instantiate a paradigm for integrating pretrained agents into LLM reasoning by projecting their internal representations into the LLM's chain of thought via lightweight adapters. Unlike tool calling, this preserves the agent's continuous action distributions and value estimates; unlike static multimodal projection, it supports evolving states through re-invocation.

    \item \textbf{Dynamic re-invocation with \texttt{InvokeAgent} (Section~\ref{sec:methodology}):} We introduce a mechanism for iterative reasoning over evolving states: the LLM generates actions, transitions the environment, and receives fresh agent encodings mid-generation. Ablations show this is the critical differentiator---removing it degrades multi-step planning by 64\% but single-step tasks by only 1--4\%.

    \item \textbf{RL-trained agent-in-the-loop reasoning (Section~\ref{sec:methodology}):} We train augmented chain-of-thought traces with GRPO, enabling the model to learn optimal agent invocation and routing strategies---including selecting among multiple agent configurations of different depths. This optimizes over full reasoning rollouts containing latent tokens, giving the RL signal a meaningful structure to improve upon.

    \item \textbf{Comprehensive empirical validation (Section~\ref{sec:experiments}):} On chess as a controlled testbed, a single LLAMIA model matches or exceeds task-specific specialists across five evaluation categories (gameplay, behavior prediction, explanation, planning, state understanding) while outperforming 30$\times$ larger LLMs with tool access. We release LLAMIA-Bench for evaluating LLM-agent integration on sequential decision tasks.
\end{enumerate}
