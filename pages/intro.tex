
\section{Introduction}
\vspace{-2mm}

% =============================================================================
% P1: The paradigm — LLMs as orchestrators of external agents
% =============================================================================

Large language models are increasingly deployed not as standalone reasoners but as orchestrators of external agents and tools~\cite{wu2023autogen,schick2023toolformerlanguagemodelsteach,shen2024hugginggpt}. Two complementary paradigms have emerged for this collaboration. The first, \textit{tool calling}, interfaces the LLM with external agents through text: the model queries an agent via a structured API, receives a symbolic response, and conditions its next reasoning step on this output~\cite{yao2022react,wu2023autogen}. Recent work has shown that reinforcement learning can teach LLMs \textit{when} and \textit{how} to invoke such tools, yielding substantial gains over supervised approaches on mathematical reasoning~\cite{toolrl2025}. The second paradigm, \textit{latent multi-agent collaboration}, replaces text-based communication between language models with continuous representation exchange---passing hidden states or KV caches between agents rather than decoded tokens~\cite{latentmas2025,hao2024training}. Both paradigms share a key assumption: the collaborating agent either communicates in natural language or shares the LLM's representational architecture.

% =============================================================================
% P2: The information bottleneck for heterogeneous agents
% =============================================================================

This assumption breaks for a large class of agents we term \textit{heterogeneous}: pretrained neural networks for game playing~\cite{silver2017mastering,monroe2024mastering}, robotic control~\cite{brohan2023rt2}, reward modeling~\cite{xu2023imagereward}, or domain-specific evaluation whose internal representations occupy fundamentally different spaces from language models. For these agents, tool calling remains the only integration path---and it creates a severe information bottleneck. Consider a chess engine: when queried via the standard UCI protocol, even an enriched response (top-$k$ moves with centipawn evaluations, win/draw/loss probabilities, principal variations) compresses the agent's full internal state into ${\sim}$10--20 tokens. The engine's latent representation actually encodes a probability distribution over all ${\sim}$30 legal moves, value estimates for future positions, and positional features capturing concepts such as king safety and pawn structure~\cite{mcgrath2022acquisition,jenner2024evidence}. None of this transfers through the text interface. The LLM then spends ${\sim}$100+ tokens parsing and reasoning about these sparse symbolic outputs---an inefficiency that compounds over multi-step interactions.

% =============================================================================
% P3: Why existing solutions don't close this gap
% =============================================================================

Prior approaches each address part of the problem but leave this gap open. \textit{Language-only fine-tuning}~\cite{feng2024chessgpt} trains LLMs on move sequences and textual annotations, operating entirely in token space without access to the agent's learned representations. Empirically, a 13B model fine-tuned this way achieves only 11\% behavior cloning accuracy where our approach achieves 55\% (Table~\ref{tab:chess_benchmarks}). \textit{RL-augmented tool calling}~\cite{toolrl2025} teaches optimal tool invocation strategies, but the communication channel remains textual---the information bottleneck persists regardless of when the tool is called. \textit{Latent multi-agent systems}~\cite{latentmas2025} enable rich continuous collaboration, but require homogeneous architectures; they cannot accommodate a 240M-parameter chess transformer whose embedding space is incompatible with the LLM's. Finally, \textit{static multimodal projection}~\cite{liu2023visual,li2023blip} encodes an input once and holds it fixed throughout generation. In sequential decision domains, this fails: the LLM's actions change the environment state, which must be re-encoded for continued reasoning.

% =============================================================================
% P4: Our approach
% =============================================================================

These observations motivate \textit{latent state internalization}: rather than communicating with heterogeneous agents through text, we project their internal representations directly into the LLM's chain of thought. A lightweight adapter $H_\phi: \mathbb{R}^{d} \rightarrow \mathbb{R}^{K \times e}$ ({<}1\% additional parameters) maps the agent's latent state into $K{=}32$ continuous tokens in the LLM's embedding space. These tokens are interleaved with language and action tokens during autoregressive generation. Crucially, an \texttt{<InvokeAgent>} mechanism enables \textit{dynamic re-encoding}: when the LLM generates an action, the environment transitions, the agent re-encodes the new state, and fresh latent tokens are appended to the reasoning trace. This closed loop supports iterative planning over evolving states without external API calls. We train these augmented reasoning traces with Group Relative Policy Optimization (GRPO), enabling the model to learn \textit{when} to invoke the agent, \textit{which} configuration to use (routing between agents of different search depths), and \textit{how} to condition its reasoning on the resulting latent tokens. Unlike prior applications of GRPO to single-step move prediction~\cite{mom2025}, our formulation optimizes over full chain-of-thought rollouts containing interleaved latent and language tokens, providing a meaningful reasoning structure for the RL signal to act on.

% =============================================================================
% P5: Why chess
% =============================================================================

We validate this paradigm on chess, chosen as a controlled testbed for three reasons. First, chess provides deterministic dynamics and 60+ years of calibrated evaluation through the Elo rating system, enabling precise measurement of planning capability. Second, superhuman pretrained agents exist (Leela Chess Zero~\cite{monroe2024mastering}, Stockfish~\cite{romstad2008stockfish}) with well-understood internal representations~\cite{jenner2024evidence,mcgrath2022acquisition}, allowing us to study what is gained by internalizing their latent states versus querying them symbolically. Third, chess offers a rich suite of evaluation tasks---gameplay, behavior cloning across skill levels, natural language commentary, instruction-conditioned planning, and state understanding---enabling comprehensive assessment of the internalization paradigm across both decision-making and decision-understanding capabilities.

% =============================================================================
% P6: Key results
% =============================================================================

We instantiate this framework as \textbf{LLAMIA} (Large Language and Action Models with Internal Agents), combining Vicuna-7B/13B~\cite{zheng2023judging} with Leela Chess Zero's BT4 network (240M parameters). Key findings:

\begin{itemize}[leftmargin=*, topsep=2pt, itemsep=2pt]
    \item \textbf{Internalization outperforms tool calling across all tasks:} LLAMIA-13B achieves 0.61 G-eval on commentary vs.\ 0.52 for GPT-4o with Lc0 tool access, and 17\% win-rate gain on instruction-conditioned planning vs.\ 10\% for LLaMA-3 405B with Lc0---while being 30$\times$ smaller (Table~\ref{tab:chess_benchmarks}).

    \item \textbf{A single model matches or exceeds task-specific specialists:} LLAMIA-13B achieves 55\% behavior cloning accuracy with one model vs.\ 53\% for Maia's 5-model ensemble trained on 12$\times$ more data, and 2279$\pm$41 Elo without search, comparable to the 270M Transformer (2299 Elo) trained on 500$\times$ more positions (Table~\ref{tab:gameplay-performance}).

    \item \textbf{Dynamic re-encoding is the critical mechanism:} Removing \texttt{<InvokeAgent>} causes 64\% degradation on conditional planning and 22\% on commentary, but only 1--4\% on single-step tasks (gameplay, move prediction). This asymmetry precisely isolates the contribution of iterative latent re-encoding to multi-step reasoning (Table~\ref{tab:invoke_ablation}).

    \item \textbf{Both training stages are necessary:} Stage~1 (projector alignment on 50M state-policy pairs) alone achieves strong gameplay (2198 Elo) but poor language tasks (0.31 G-eval). Stage~2 without Stage~1 is unstable and plateaus 190 Elo below the full system. The staged approach leverages abundant synthetic data for stable initialization before instruction-tuning (Table~\ref{tab:stage_ablation}).
\end{itemize}

\vspace{-1mm}
\subsection{Contributions}
\vspace{-1mm}

\begin{enumerate}[leftmargin=*, labelsep=0.5em, topsep=2pt, itemsep=2pt]
    \item \textbf{Latent state internalization for heterogeneous agents (Section~\ref{sec:methodology}):} We formalize and instantiate a paradigm for coupling LLMs with pretrained non-language agents by projecting their internal representations into the LLM's reasoning trace via learned adapters. Unlike tool calling, this preserves continuous policy and value information. Unlike latent multi-agent systems, this bridges architecturally heterogeneous models through a lightweight projector.

    \item \textbf{Dynamic re-encoding with \texttt{InvokeAgent} (Section~\ref{sec:methodology}):} We introduce a mechanism for iterative reasoning over evolving states: the LLM generates actions, transitions the environment, and receives fresh agent encodings mid-generation. Ablations confirm this is the critical differentiator---removing it causes 64\% degradation on multi-step planning but only 1--4\% on single-step tasks.

    \item \textbf{GRPO-trained agent-in-the-loop reasoning (Section~\ref{sec:methodology}):} We train augmented chain-of-thought traces with GRPO, enabling the model to learn optimal agent invocation strategies---including routing between multiple agent configurations of different depths. This optimizes over full reasoning rollouts containing latent tokens, rather than bare action prediction.

    \item \textbf{Comprehensive empirical validation (Section~\ref{sec:experiments}):} On chess as a controlled testbed, a single LLAMIA model matches or exceeds task-specific specialists across five evaluation categories while outperforming 30$\times$ larger LLMs with tool access. We release LLAMIA-Bench for evaluating LLM-agent integration on sequential decision tasks.
\end{enumerate}
