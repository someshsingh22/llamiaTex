
\section{Introduction}
\vspace{-2mm}
Large language models (LLMs) are increasingly deployed to collaborate with domain
agents to solve tasks that demand both general reasoning and
domain-specific expertise~\cite{wu2023autogen,agentverseXXX}.
Today this collaboration is primarily mediated through text: the LLM invokes an agent, receives a
natural-language description of its output, and reasons over that
description to decide subsequent
actions~\cite{yao2022react,schick2023toolformerlanguagemodelsteach}.
Such \emph{verbalized} collaboration is effective when both agents are themselves language models as they are trained in the same input space. However, in many domains like game playing, robotics, autonomous driving, the
strongest available agents are not language models but pretrained neural networks whose expertise
is encoded in continuous representations: policy distributions, value
estimates, and learned features~\cite{silver2017mastering,monroe2024mastering,brohan2023rt2}. This dichotomy instigates a fundamental question:
\begin{quote}
\emph{How can LLMs effectively collaborate and jointly reason with pretrained non-language agents?}    
\end{quote}

This problem is central to the LLM based agents paradigm. As LLMs increasingly serve as general purpose orchestrators, the depth of their collaboration with domain experts determines what these systems can ultimately achieve. Effective collaboration has the potential to unlock creative applications that neither agent achieves alone.
Consider chess: LLMs have been trained on more chess literature than most grandmasters will study in a lifetime, yet they struggle to understand it at the level of an amateur\cite{XXXHLE or ChessBench etc results}, and despite surpassing grandmasters decades ago \cite{campbell2002deep} pretrained engines like AlphaZero~\cite{silver2017mastering} remain difficult to interpret without expert intervention, and require extensive manual analysis to explain the rationale behind a move. 
Consequently, the most valued applications like imitating human gameplay\cite{MAIACHessWebsite}, understanding and explaining the rationale behind a move\cite{CHESS.com}, strategizing against an opponent's known weaknesses, commentating on a game, or assessing which puzzles are difficult or interesting remain unsolved. These creative applications demand both, the engine's deep positional understanding and the ability to reason over human intent and perception in natural language. Improving how LLMs collaborate with domain agents would not only unlock these applications but make them scalable and accessible.

These tasks are well-defined, practically relevant, and have established evaluation protocols. Chess has consequently become a model system for studying language model capabilities across planning\cite{XXX}, search~\cite{diffusearch2025}, reasoning~\cite{XXX}. Prior approaches each address part of the problem but leave this gap open. \textit{Language-only fine-tuning}~\cite{feng2024chessgpt} trains LLMs on move sequences and textual annotations, operating entirely in token space without access to the agent's learned representations. Empirically, a 13B model fine-tuned this way achieves only 11\% behavior cloning accuracy where our approach achieves 55\% (Table~\ref{tab:chess_benchmarks}). \textit{RL-augmented tool calling}~\cite{toolrl2025} teaches optimal tool invocation strategies, but the communication channel remains textual---the information bottleneck persists regardless of when the tool is called. \textit{Latent multi-agent systems}~\cite{latentmas2025} enable rich continuous collaboration, but require homogeneous architectures; they cannot accommodate a 240M-parameter chess transformer whose embedding space is incompatible with the LLM's. \somesh{Let's leave this last part, it is not standard for LLM and Chess or Games etc Literature}Finally, \textit{static multimodal projection}~\cite{liu2023visual,li2023blip} encodes an input once and holds it fixed throughout generation. In sequential decision domains, this fails: the LLM's actions change the environment state, which must be re-encoded for continued reasoning.

% P4: Our approach
% =============================================================================

These observations motivate \textit{latent state internalization} \somesh{No, first of all this is a novel concept, we need to moitvate why this is needed and what it is very clearly}: rather than communicating \somesh{communicating trivializes it, use verbalizing as text}with heterogeneous agents through text, we project their internal representations directly into the LLM's chain of thought. A lightweight adapter $H_\phi: \mathbb{R}^{d} \rightarrow \mathbb{R}^{K \times e}$ ({<}1\% additional parameters) maps the agent's latent state into $K{=}32$ continuous tokens in the LLM's embedding space. These tokens are interleaved with language and action tokens during autoregressive generation. Crucially, an \texttt{<InvokeAgent>} mechanism enables \textit{dynamic re-encoding}: when the LLM generates an action, the environment transitions, the agent re-encodes the new state, and fresh latent tokens are appended to the reasoning trace. This closed loop supports iterative planning over evolving states without external API calls. We train these augmented reasoning traces with Group Relative Policy Optimization (GRPO), enabling the model to learn \textit{when} to invoke the agent, \textit{which} configuration to use (routing between agents of different search depths), and \textit{how} to condition its reasoning on the resulting latent tokens.

We instantiate this framework as \textbf{LLAMIA} (Large Language and Action Models with Internal Agents), combining Vicuna-7B/13B~\cite{zheng2023judging} with Leela Chess Zero's BT4 network (240M parameters). Key findings: \somesh{Key findings should be in the paragraph before contributions, right now it looks like contribution itself,}

\begin{itemize}[leftmargin=*, topsep=2pt, itemsep=2pt]
    \item \textbf{Internalization outperforms tool calling across all tasks:} LLAMIA-13B achieves 0.61 G-eval on commentary vs.\ 0.52 for GPT-4o with Lc0 tool access, and 17\% win-rate gain on instruction-conditioned planning vs.\ 10\% for LLaMA-3 405B with Lc0---while being 30$\times$ smaller (Table~\ref{tab:chess_benchmarks}).

    \item \textbf{A single model matches or exceeds task-specific specialists:} LLAMIA-13B achieves 55\% behavior cloning accuracy with one model vs.\ 53\% for Maia's 5-model ensemble trained on 12$\times$ more data, and 2279$\pm$41 Elo without search, comparable to the 270M Transformer (2299 Elo) trained on 500$\times$ more positions (Table~\ref{tab:gameplay-performance}).

    \item \textbf{Dynamic re-encoding is the critical mechanism:} Removing \texttt{<InvokeAgent>} causes 64\% degradation on conditional planning and 22\% on commentary, but only 1--4\% on single-step tasks (gameplay, move prediction). This asymmetry precisely isolates the contribution of iterative latent re-encoding to multi-step reasoning (Table~\ref{tab:invoke_ablation}).

    \item \textbf{Both training stages are necessary:} Stage~1 (projector alignment on 50M state-policy pairs) alone achieves strong gameplay (2198 Elo) but poor language tasks (0.31 G-eval). Stage~2 without Stage~1 is unstable and plateaus 190 Elo below the full system. The staged approach leverages abundant synthetic data for stable initialization before instruction-tuning (Table~\ref{tab:stage_ablation}).
\end{itemize}

\vspace{-1mm}
\subsection{Contributions}
\vspace{-1mm}

\begin{enumerate}[leftmargin=*, labelsep=0.5em, topsep=2pt, itemsep=2pt]
    \item \textbf{Latent state internalization for heterogeneous agents (Section~\ref{sec:methodology}):} We formalize and instantiate a paradigm for coupling LLMs with pretrained non-language agents by projecting their internal representations into the LLM's reasoning trace via learned adapters. Unlike tool calling, this preserves continuous policy and value information. Unlike latent multi-agent systems, this bridges architecturally heterogeneous models through a lightweight projector.

    \item \textbf{Dynamic re-encoding with \texttt{InvokeAgent} (Section~\ref{sec:methodology}):} We introduce a mechanism for iterative reasoning over evolving states: the LLM generates actions, transitions the environment, and receives fresh agent encodings mid-generation. Ablations confirm this is the critical differentiator---removing it causes 64\% degradation on multi-step planning but only 1--4\% on single-step tasks.

    \item \textbf{GRPO-trained agent-in-the-loop reasoning (Section~\ref{sec:methodology}):} We train augmented chain-of-thought traces with GRPO, enabling the model to learn optimal agent invocation strategies---including routing between multiple agent configurations of different depths. This optimizes over full reasoning rollouts containing latent tokens, rather than bare action prediction.

    \item \textbf{Comprehensive empirical validation (Section~\ref{sec:experiments}):} On chess as a controlled testbed, a single LLAMIA model matches or exceeds task-specific specialists across five evaluation categories while outperforming 30$\times$ larger LLMs with tool access. We release LLAMIA-Bench for evaluating LLM-agent integration on sequential decision tasks.
\end{enumerate}
